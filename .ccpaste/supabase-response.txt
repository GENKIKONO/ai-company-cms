A で進めてください。最小変更でビルドブロッカーを即時解消できます。

適用パッチ（@typescript-eslint 系を全削除）
.eslintrc.json の rules から @typescript-eslint 系ルールを全て取り除きます。既に3つ削除済みなので、no-explicit-any も削除します。

"rules"
: {
    
"react/no-unescaped-entities"
: 
"off"
,
    
"react-hooks/exhaustive-deps"
: 
"warn"
,
    
"@next/next/no-img-element"
: 
"warn"
,
    
"no-console"
: 
"warn"
,
    
"prefer-const"
: 
"warn"
,
    
"no-var"
: 
"error"
,
    
"eqeqeq"
: 
"error"
,
-   
"@typescript-eslint/no-explicit-any"
: [
"warn"
, { 
"ignoreRestArgs"
: 
false
 }]
+   
// note: @typescript-eslint rules are disabled for now

  },

適用後に以下を実行してください。

npm run typecheck
npm run build
ビルドが通過したら、認証付きAPIの最終確認→コミット→デプロイの順に進めましょう。
後続で型安全を強化したくなった場合は、別タスクで以下の導入を提案します（負債にならない形で段階導入可能）。

extends に "plugin:@typescript-eslint/recommended"
parser: "@typescript-eslint/parser"
必要に応じて parserOptions.project を設定し、type-aware ルールを少しずつ再有効化（no-unsafe-* など）。*
